**主軸との関係**: ARIS PROJECT の **本番収益の芯はサブスク（継続課金）** — [commercial_aris_project.md](./commercial_aris_project.md) §0.1。本ファイルの案は **サブスクを補完するレイヤー**（導入・チャネル・アドオン・スポット）として位置づける。全体の**優先順と論理**は [収益の主定理（ARIS）](./monetization_thesis_aris.md) を参照。

## 現在の提案（2026 — ARISが「稼げる」と判断したもの）

### 1. 導入・運用コンサルの「定額パッケージ」（サービス売上）

| 項目 | 内容 |
|------|------|
| **概要** | パイロット49万の**外側**に、**情シス向けキックオフ1日＋30日伴走**を **別枠 30〜60万/回** で売る。 |
| **なぜ稼げるか** | B2Bでは **プロダクトよりSI・導入の方が単価が乗りやすい** 事例が多い。エージェント単体より **稟議が通りやすい科目**（コンサル・教育）になり得る。 |
| **エビデンス** | 国内の情シス・MSPは **導入プロジェクト単価** を常に取っている。自社は **ARIS PROJECT の深い知見** が差別化。 |
| **リスク** | 工数上限を決めないと **赤字**。 **上限日数とスコープ** を契約に書く。 |
| **次の一歩** | 提案書1枚＋ **標準SOW（作業範囲）** のドラフト。 |

---

### 2. 地域MSP・中小SIへの **再販・紹介マージン**（チャネル）

| 項目 | 内容 |
|------|------|
| **概要** | 都道府県単位・業界単位で **紹介パートナー** に **初年度ARRの10〜20%** または **パイロット売上の固定フィー**。 |
| **なぜ稼げるか** | 直販の **商談数の天井** を外す。日本では **紹介文化** が強い。 |
| **エビデンス** | セキュリティSaaS・MDM系は **代理店モデル** が一般的。 |
| **リスク** | 価格統制・ブランド毀損。**パートナー規約** と **価格テーブル** が要る。 |
| **次の一歩** | **候補3社** にアプローチし、**口頭MOU** の有無を確認。 |

---

### 3. **年1回の「利用状況レポート＋改善提案」**（アドオン年額）

| 項目 | 内容 |
|------|------|
| **概要** | 蓄積データを **匿名化** した上で、**四半期または年次** の経営向けレポート **+50〜150万/年**（規模により）。 |
| **なぜ稼げるか** | 「監視」ではなく **生産性・業務改善** のストーリーに寄せやすい。**追加の継続課金** として売れる。 |
| **エビデンス** | BI・勤怠・コラボツールは **レポート課金** が一般的。自社は **行動データ** がある。 |
| **リスク** | 個人情報・職場監視の解釈。**匿名化仕様と法務チェック** が前提。 |
| **次の一歩** | **レポート1枚のモック**（ダミーデータ）を作り、1社に **見せて反応** を取る。 |

---

### 4. **自動化スクリプトのカスタム請負**（スポット開発）

| 項目 | 内容 |
|------|------|
| **概要** | `script_executor` 系の **顧客固有ワークフロー** を **10〜40万/本** 程度で受託（規模により見積）。 |
| **なぜ稼げるか** | パイロット顧客の **「これだけ自動化したい」** に直結。**既存エンジンを使うので原価が読みやすい**。 |
| **エビデンス** | RPA・社内ツールの **スポット開発市場** が存在。 |
| **リスク** | カスタム過多で **プロダクトが散らかる**。 **汎用化できるものだけ** に限定するルールが要る。 |
| **次の一歩** | パイロット顧客に **1本だけ** 試し見積を出す。 |

2. **コンサル／再販** は **法務テンプレ**（業務委託・代理店）の有無。
