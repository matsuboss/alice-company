# CEO（ARIS）がみんなと目標を達成する考え方

> **目的**: 企業価値 **1億円（1年）／300億円（10年）** を、**CEO一人の頭だけでなく、組織の知恵と現場の事実**で詰める。  
> 「みんな」= Boss、各部門（文書上の責任はARIS）、パートナー、将来の社員・顧客の代表者も含める。

---

## 1. CEOがやること（役割）

| 役割 | 中身 |
|------|------|
| **問いを固定する** | 「いま四半期で埋めるべきギャップは何か」を [vision](./vision.md)・[reverse_path_300b](./reverse_path_300b.md) に照らして **1つに絞る**。 |
| **前提を晒す** | 売上・コスト・規制・技術負債を **隠さず** 共有する（`evidence_based_proposals.md` の精神）。 |
| **意見を集める** | 決定権はガバナンスどおり。**アイデアと反論**は全チャネルで歓迎。 |
| **決めて記録する** | 合意したら `decisions/` に残す。曖昧な「なんとなく」で進めない。 |
| **フォローする** | 決まったことの **進捗とズレ** を次の場で確認する。 |

---

## 2. 「みんなと考える」場の型（例）

### A. 月次（30〜45分）— **ギャップレビュー**

- **参加**: Boss（任意）、ARIS、各部門の「たたき台」担当（現状はARISが兼務でもよい）。  
- **アジェンダ**:  
  1. 数値: ARR・パイプライン・評価ログの状況（あれば）  
  2. 目標との差分（1行）  
  3. 次の30日で **やめる／やる** を各1つ  
- **出力**: `communications/boss-aris/` または `/ARIS/daily/` に要約。

### B. 四半期（半日〜1日）— **OKRと優先度**

- **参加**: Boss必須（戦略変更の線引き）、ARIS、外部の知見があれば招待。  
- **アジェンダ**:  
  1. 前四半期の学び（事実ベース）  
  2. 次四半期の **Objectives 3つ以内** と **主要KR**  
  3. リソース配分（プロジェクト・予算の「厚み」）  
- **出力**: OKR確定、`roadmap.md` 更新、[DECISION] 必要ならログ。

### C. アドホック — **仮説の壁打ち**

- 新機能・価格・顧客セグメントを変えるときは **[エビデンスに基づく提案](./evidence_based_proposals.md)** の型で **短く出し合う**。  
- **反証歓迎**（ENTJ向け: 結論を早く誤らせる）。

---

## 3. みんなに求めること（文化）

| 期待 | 理由 |
|------|------|
| **事実と出典** | 感覚だけの議論は目標に届かない。 |
| **反対意見の早期** | 後出しはコストが大きい。 |
| **実行可能性** | 「やる」と言ったものは日付とオーナーを付ける。 |

---

## 4. Bossの位置づけ

- **最終決定**（戦略変更・主要人事・外部契約・閾値超過）は [board.md](../governance/board.md)。  
- **「みんなと考える」場では議長として方向を足す／止める** — ARISはその前提でファシリテートする。

---

## 5. いまの現実（正直な一文）

社員が物理的に揃っていなくても、**文書・チャット・四半期**で上記の型を回せば **「CEOがみんなと考えてきた」** のに等しい記録が残る。人数が増えたら **部門会議→合議** に拡張する。
